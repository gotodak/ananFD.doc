ユーザ認証・アクセス制御

Zend_Auth－ユーザ認証

Zend_Auth は、ユーザ認証を行うことができるコンポーネントです。 認証とは､アクセスしてきたユーザが正規ユーザか否かをチェックする機能です。
正規ユーザでは無い場合は画面を見せない、などといった「アクセス制御」の機能は一切持っていませんので、注意してください。

Zend_Auth は、認証方式として次のような方法を提供しています。

Zend_Auth が提供している認証方式
認証方式 概要
データベースを利用した認証 データベースに保存されている ID とパスワードで認証を行う
独自実装による認証 ファイルなどに保存されている ID とパスワードで認証を行う
Basic 認証を利用した認証 Web サーバのBasic 認証機能を利用する
ダイジェスト認証を利用した認証 Web サーバのダイジェスト認証機能を利用する
OpenID を利用した認証 OpneID サーバの認証機能を利用する
LDAP を利用した認証 LDAP サーバの認証機能を利用する

Zend_Acl－アクセス制御リスト

Zend_Acl は、アクセス制御を行うコンポーネントです。
アクセス制御とは、そのユーザがこのページにアクセスする権限があるかをチェックをし、 適切に処理を行う機能のことです。
Zend_Acl には、「リソース」と「ロール」という２つの概念があります。
「リソース」はアクセス制御を行う対象になるオブジェクトのことで、 ＷＥＢページそのものや登録などの機能のことを指します。
「ロール」は、リソースに対してアクセスを要求するオブジェクトのことで、 アクセス権限の種類を定義しているもののことです。

リソースとロール
名称 概要 例
リソース アクセス制御を行う対象になるオブジェクト ＷＥＢページそのもの、登録などの機能
ロール リソースに対してアクセスを要求するオブジェクト アクセス権限の種類を定義しているもの

アクセス制御の定義

今回は、ロールの種類によってアクセス制御を行っていますが、 ロールの中でさらに細かい設定を追加することもできます。
例えば、「閲覧は許可するが、新規登録は許可しない」などの設定が可能です。
今回、各画面の機能における仕様（ユーザごとアクセス制御の詳細仕様…）が分からない部分が多々あり、
そのため、ロール、リソースを細分化してアクセス制御の仕様変更柔軟に対応できるようにするため、 アクセス制御の定義は使用しておりません。

想定（アクセス制御）
ロール 利用権限
guest 閲覧のみ
staff 閲覧のみ
teacher 閲覧・新規作成
manager 閲覧・新規作成・編集
master 閲覧・新規作成・編集・削除
admin 閲覧・新規作成・編集・削除・復活

Zend_Session－セッション管理

Web プログラミングにおいて、最もよく利用されている機能の１つに「セッション」があります。
サーバプログラムでは、クライアントを識別するために、 セッション識別子と呼ばれる一意なコードをサーバ／クライアントでやりとりして、
各ユーザを個別に管理できるようにしています。
