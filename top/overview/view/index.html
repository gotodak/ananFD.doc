Zend_View－ビュー

Zend_View の概要

「Zend_View」は画面表示に関する処理を担当するコンポーネントで、ＭＶＣモデルの中で「Ｖ」の役割を担います。
Zend_Framework では、表示に関する機能をすべて Zend_View に集約させているため。
ほとんどのアプリケーションでこのコンポーネントを利用することになります。
また、ＭＶＣモデルでアプリケーションを作成すると。デフォルト設定で自動的に動作するようになっています。

ちなみに、Zend_View は非常に汎用的に設計されているため、「Smarty」などの外部テンプレートエンジンを代わりに利用することも可能です。

Zend_View では、「ビュースクリプト」というファイルを作成し、そこに定義された変数に値をセットした後に
「ビュースクリプト」を実行することにより、画面表示を実現しています。

ビュースクリプトについて

ビュースクリプトの正体は、単なるＰＨＰファイルです。
そのため、特に新しい知識は不要ですが、いくつかのポイントは押さえておく必要があります。

  • ＨＴＭＬにいくつかのＰＨＰコードが埋め込まれた形式であること
  • ＰＨＰコードでは制御構造文と変数の加工と出力しか行わないこと
  • 特別な理由がない限り拡張子を「phtml」にすること

つまり、ＰＨＰファイルであるといえ、表示に関する最低限のことしか行わないようにしなければなりません。

コントローラ側の処理

ビュースクリプトだけでは画面表示できません。 実際にはコントローラからビュースクリプトをロードして表示させることになります。
ビュースクリプト内の変数に文字列を代入するのもコントローラ側の役目です。
自動レンダリングモードが有効か無効かによって処理の手順が異なりますので、それぞれの場合について説明します。

自動レンダリングモードが有効の場合

自動レンダリングモードが有効（デフォルト設定）になっている場合は、
Zend_View のほとんどのパラメータが自動的に設定され、コーディングの手間がかなり軽減されるらしい？
利用手順は次の通りです。

  1. 決められたディレクトリ構成でビュースクリプトを設置する
  2. Zend_View の文字コードを設定する
  3. ビュースクリプト内の変数にデータを設定する

自動レンダリングモードでのコントローラ側の処理は以上で完了です。
あとはアクションの処理の終了時に Zend_View のレンダリング処理が実行されます。

自動レンダリングモードが有効な場合の、コントローラとビュースクリプトのサンプルを以下に表示します。

  14.             <?php foreach ($this->person as $value): ?>
  15.             <tr>
  16.                 <td> <?php echo $this->escape($value['name']); ?></td>
  17.                 <td> <?php echo $this->escape($value['tel']); ?></td>
  18.                 <td> <?php echo $this->escape($value['address']); ?></td>

自動レンダリングモードが無効の場合

自動レンダリングモードが無効になっている場合には、他のコンポーネントを利用する場合と同様に、
コンポーネントのロードからレンダリングの処理まで。すべての行程をプログラミングする必要があります。
自動レンダリングモードが有効な場合と比べると手間は増えますが、細かな設定や調整を行いたい場合は逆にこちらの方が有利になります。
利用手順は次の通りです。

  1. インスタンスを生成する
  2. ビュースクリプトを設置するディレクトリを設定する
  3. 設定したディレクトリにビュースクリプトを設置する
  4. Zend_View の文字コードを設定する
  5. ビュースクリプト内の変数にデータを設定する
  6. レンダリング処理を行う
  7. レンダリング結果をレスポンスオブジェクトにセットする

ビューヘルパー

ビューヘルパーとは、ビュースクリプトを作成する際に、便利に利用できるモジュール群です。
通常通り記述するのは面倒なものを、自動的に出力してくれる機能や、 複雑な繰り返し処理などを簡単に記述できる機能など、提供されている機能は様々です。
現在、標準で提供されているビューヘルパーは次の通りです。

標準のヘルパー
ヘルパー 概要
form ヘルパー フォーム情報を埋め込む
Action ヘルパー ビュースクリプトの中からコントロールのアクションを実行して結果を埋め込む
Partial ヘルパー ビュースクリプトの中から別のビュースクリプトを埋め込む
Placeholder ヘルパー 指定文字列に対する置換処理を行う
Doctype ヘルパー Doctype 宣言を埋め込む
HeadLink ヘルパー CSS や RSS などのリンク要素を埋め込む
HeadMedia ヘルパー META 情報を埋め込む
HeadScript ヘルパー JavaScript などのスクリプト情報を埋め込む
HeadStyle ヘルパー CSS 情報を埋め込む
HeadTitle ヘルパー タイトル情報を埋め込む
HTML オブジェクトヘルパー Flash や動画などのオブジェクト情報を埋め込む
inlineScript ヘルパー JavaScript などのスクリプト自体を埋め込む
JSON ヘルパー JSON のヘッダ情報を出力する
翻訳ヘルパー Zend_Translate と連動して翻訳データを埋め込む

form ヘルパー

form ヘルパーは、HTML のフォーム関連のタグを埋め込んでくれるビューヘルパーです。
これを使用するとことで、プログラムと密接に関連するフォームの記述を、分かりやすく簡潔に書けます。

メソッド一覧
メソッド 概要
form() フォームの開始タグと終了タグを埋め込む
fromText() テキストボックスを埋め込む
formTextarea() テキストエリアを埋め込む
formPassword パスワード入力ボックスを埋め込む
formRadio() ラジオボタンを埋め込む
formCheckbox() チェックボックスを埋め込む
formMultiCheckbox() チェックボックスを配列形式で埋め込む
formSelect() セレクトボックスを埋め込む
formFile() ファイル選択ボックスを埋め込む
formHidden() hidden 要素を埋め込む
formLabel() ラベル要素を埋め込む
formButton() ボタンを埋め込む
formReset() リセットボタンを埋め込む
formSubmit() 送信ボタンを埋め込む

HTML オブジェクトヘルパー

HTMLファイルを作成する際、最も難解なタグの１つが「OBJECT」タグです。
Flashや動画などを表示させるには、「OBJECT」タグを使わなくてはなりませんが、
複雑な記述が必要になります。これを簡単に記述できるようにするのが、HTML オブジェクトヘルパーです。

メソッド一覧
メソッド 概要
htmlObject() OBJECT タグを埋め込む汎用メソッド
htmlFlash() FLASH 表示用のタグを埋め込む
htmlQuicktime() Quicktime形式の動画表示用のタグを埋め込む
